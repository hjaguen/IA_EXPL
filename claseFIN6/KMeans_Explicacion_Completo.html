Qu es K-Means?

K-Means es un algoritmo de clustering no supervisado que organiza datos en K grupos o clusters. Su objetivo es minimizar la distancia entre los puntos y su centroide (punto central del grupo).

Pasos del algoritmo:

  1. Elegir K centroides iniciales aleatorios.
  2. Asignar cada punto al centroide ms cercano.
  3. Recalcular los centroides como el promedio de los puntos asignados.
  4. Repetir los pasos 2 y 3 hasta que los centroides no cambien significativamente.

K-Means es eficiente y til para segmentar datos sin necesidad de etiquetas previas.

El objetivo es minimizar la inercia, que mide qu tan cerca estn los puntos de sus respectivos centroides.

Parmetro importante:

  1. n_init: nmero de veces que el algoritmo se ejecutar con diferentes centroides iniciales.
  2. K-Means selecciona los mejores resultados segn la inercia mnima obtenida.
  3. Se recomienda especificar n_init explcitamente para evitar advertencias y mejorar estabilidad.

n_init= Nmero de veces que el algoritmo K-Means se reinicia con diferentes centroides aleatorios.

Cada una de esas veces, el algoritmo:

  • Elige k centroides al azar.
  • Ejecuta el clustering completo.
  • Calcula la inercia final.
  • Luego de todas las repeticiones, se elige el mejor resultado, es decir, el que tenga menor inercia (mejores agrupamientos).

El algoritmo converge: K-Means repetir los pasos de:

  1. Asignar puntos al centroide ms cercano.
  2. Recalcular los centroides.

Hasta que:

  1. Los centroides no cambien significativamente (convergencia), o
  2. Se alcance el lmite de 300 iteraciones.
  3. Entonces, si no hay convergencia antes, el algoritmo se detiene a la iteracin 300 como mximo

Generacin de datos

Usamos make_blobs para crear un conjunto de datos artificial que simula agrupaciones naturales.

make_blobs genera conjuntos de datos sintticos que simulan agrupaciones (blobs) tiles para probar algoritmos de clustering como K-Means.

Parmetros:

  1. n_samples: nmero total de puntos de datos a generar
  2. centers: nmero de centros de cluster (o coordenadas especficas de los centros)
  3. n_features: nmero de caractersticas (dimensiones)
  4. random_state: semilla para asegurar resultados reproducibles

Qu es la Inercia?

La inercia es la suma de las distancias cuadradas desde cada punto hasta su centroide. Cuanto menor es la inercia, ms compactos estn los clusters.

Importante: por s sola no indica si el modelo es bueno. Es necesario compararla con diferentes valores de k.

In[19]:

# La inercia es la suma de las distancias cuadradas de cada punto al centro de su cluster.
# Sirve como una medida de qu tan compactos son los clusters formados.
# Cuanto menor sea la inercia, mejor es la agrupacin (los puntos estn ms cerca de su centro).

Mtodo del Codo para determinar el nmero ptimo de clusters

Se ejecuta K-Means con distintos valores de k y se grafica la inercia. El "codo" en la curva muestra el punto a partir del cual aumentar k ya no mejora significativamente la agrupacin.

In[22]:
# Mtodo del Codo para elegir el nmero ptimo de clusters
# Se entrena el modelo con diferentes valores de k (nmero de clusters)
# y se grafica la inercia correspondiente. El punto donde la inercia deja de disminuir rpidamente
# (donde se forma un "codo" en la grfica) es un buen candidato para el nmero ptimo de clusters.
# Esto nos ayuda a decidir cuntos clusters usar sin que el modelo sea innecesariamente complejo.
inercia = []
k_range = range(1, 11)
for k in k_range:
    modelo = KMeans(n_clusters=k, random_state=42, n_init=10)
    modelo.fit(X)
    inercia.append(modelo.inertia_)

plt.figure(figsize=(10, 6))
plt.plot(k_range, inercia, marker='o')
plt.title('Mtodo del Codo para Seleccin de k')
plt.xlabel('Nmero de Clusters (k)')
plt.ylabel('Inercia')
plt.xticks(k_range)
plt.grid(True)
plt.show()

Interpretacin del resultado del mtodo del codo

El punto donde la curva de inercia cambia bruscamente de pendiente (forma un "codo") representa un buen equilibrio entre simplicidad y agrupamiento efectivo. Si el codo aparece en k = 4, ese nmero es adecuado para los datos.
