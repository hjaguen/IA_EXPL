🤖 Chatbot Educativo con Naive Bayes (modo consola)¶

Este proyecto demuestra cómo crear un chatbot educativo utilizando procesamiento de lenguaje natural (NLP) y el algoritmo de clasificación Naive Bayes.

El objetivo es asistir a estudiantes, docentes y padres con respuestas automáticas en un entorno virtual primario. Se realiza todo el desarrollo paso a paso, desde el preprocesamiento hasta la clasificación y la conversación.

Temas cubiertos:

  • Ciencia de datos
  • Procesamiento de texto
  • Algoritmo de Naive Bayes
  • Aplicación educativa en consola

🧹 Paso 3: Preprocesamiento del texto¶

La función limpiar_frase normaliza los textos eliminando tildes, convirtiendo a minúsculas, tokenizando, eliminando stopwords y aplicando stemming para reducir palabras a su raíz.

In [5]:
def limpiar_frase(frase):
    #Esto descompone los caracteres Unicode en sus componentes básicos. or ejemplo: "á" se convierte en "a" + "́" (letra más tilde)
    frase = unicodedata.normalize("NFD", frase) 
    frase = frase.encode("ascii", "ignore").decode("utf-8") 
    # elimina todos los caracteres no ASCII (como tildes, eñes, etc.).
    
    frase = frase.lower()
    tokens = word_tokenize(frase, language='spanish')
    return [
        stemmer.stem(p) for p in tokens
        if p.isalpha() and p not in stopwords_es and len(p) > 2 
        #Esa línea filtra palabras, quedándose solo con aquellas que contienen solo letras, no son stopwords y tienen más de dos caracteres.
    ]

🧠 Paso 5: Clasificador Naive Bayes¶

! El algoritmo Naive Bayes se basa en la fórmula:

[ P(c|x) \propto P(c) \prod_{i=1}^n P(x_i|c) ]

Donde:

  • (P(c)): probabilidad de la categoría.
  • (P(x_i|c)): probabilidad de la palabra dado la categoría.

Utilizamos logaritmos para evitar valores pequeños y aplicamos suavizado de Laplace para evitar divisiones por cero.

Mejoras del modelo¶

🧠 ¿Para qué sirve log(P(C)) en Naive Bayes?¶

Cuando aplicas Naive Bayes, originalmente usas esta fórmula:

Pero multiplicar muchas probabilidades pequeñas (por ejemplo, 0.01 × 0.03 × 0.05) puede generar un número tan pequeño que se convierte en 0 por limitaciones del computador esto se llama underflow numérico.

🧮 ¿Cómo se soluciona?¶

Usando logaritmos. Gracias a la propiedad:

Podemos reescribir Naive Bayes así:

✅ Entonces, ¿para qué es log(P(C))?¶

Evita errores de precisión por números muy pequeños.

Convierte multiplicaciones en sumas, lo cual es más rápido y estable.

Forma parte del cálculo total de la probabilidad de la clase.

categorias = list(diccionario_cat.keys())
vocabulario = set(diccionario_total.keys())
tam_vocabulario = len(vocabulario)

def clasificar(frase):
    palabras = limpiar_frase(frase)   #limpia  la frase  # ← Preprocesamiento, se obtiene {x1, x2, ..., xn}
    puntajes = {}
    for categoria in categorias:
        total_frases = sum(conteo_frases.values())
        prob_categoria = conteo_frases[categoria] / total_frases  # ← Aplica P(C)
        log_prob = math.log(prob_categoria)   # ← Usa log(P(C))
        
        #  👉 DESDE AQUI SE PLICAN CALCULOS ANTERIORES
        total_palabras_cat = sum(diccionario_cat[categoria].values())
        for palabra in palabras:
            frecuencia = diccionario_cat[categoria][palabra]
            prob_palabra = (frecuencia + 1) / (total_palabras_cat + tam_vocabulario) # suavizado de laplace
            log_prob += math.log(prob_palabra)  # logaritmo de suavizado de laplace
        puntajes[categoria] = log_prob
    print( max(puntajes, key=puntajes.get) ) # imprime la categoria 
    return max(puntajes, key=puntajes.get)   # 👉 Finalmente, selecciona la categoría con mayor probabilidad (log-verosimilitud)

def iniciar_chatbot():
    print("🧠 Chatbot Educativo (escribe 'salir' para terminar)")
    while True:
        pregunta = input("Tú: ")
        if pregunta.lower() == "salir":
            print("Chatbot: ¡Hasta luego!")
            break
        respuesta = responder(pregunta)
        print(f"Chatbot: {respuesta}\n")
