La funcin limpiar_frase normaliza los textos eliminando tildes, convirtiendo a minsculas, tokenizando, eliminando stopwords y aplicando stemming para reducir palabras a su raz.

In[5]:
def limpiar_frase(frase):
    #Esto descompone los caracteres Unicode en sus componentes bsicos. or ejemplo: "" se convierte en "a" + "" (letra ms tilde)
    frase = unicodedata.normalize("NFD", frase) 
    frase = frase.encode("ascii", "ignore").decode("utf-8") 
    # elimina todos los caracteres no ASCII (como tildes, ees, etc.).
    
    frase = frase.lower()
    tokens = word_tokenize(frase, language='spanish')
    return [
        stemmer.stem(p) for p in tokens
        if p.isalpha() and p not in stopwords_es and len(p) > 2 
        #Esa lnea filtra palabras, quedndose solo con aquellas que contienen solo letras, no son stopwords y tienen ms de dos caracteres.
    ]

Paso 5: Clasificador Naive Bayes

! El algoritmo Naive Bayes se basa en la frmula:

[ P(c|x) \propto P(c) \prod_{i=1}^n P(x_i|c) ]

Donde:

  • (P(c)): probabilidad de la categora.
  • (P(x_i|c)): probabilidad de la palabra dado la categora.

Utilizamos logaritmos para evitar valores pequeos y aplicamos suavizado de Laplace para evitar divisiones por cero.

Mejoras del modelo

Para qu sirve log(P(C)) en Naive Bayes?

Cuando aplicas Naive Bayes, originalmente usas esta frmula:

Pero multiplicar muchas probabilidades pequeas (por ejemplo, 0.01 0.03 0.05) puede generar un nmero tan pequeo que se convierte en 0 por limitaciones del computador esto se llama underflow numrico.

Cmo se soluciona?

Evita errores de precisin por nmeros muy pequeos.

Convierte multiplicaciones en sumas, lo cual es ms rpido y estable.

categorias = list(diccionario_cat.keys())
vocabulario = set(diccionario_total.keys())
tam_vocabulario = len(vocabulario)

def clasificar(frase):
    palabras = limpiar_frase(frase)   #limpia  la frase  #  Preprocesamiento, se obtiene {x1, x2, ..., xn}
    puntajes = {}
    for categoria in categorias:
        total_frases = sum(conteo_frases.values())
        prob_categoria = conteo_frases[categoria] / total_frases  #  Aplica P(C)
        log_prob = math.log(prob_categoria)   #  Usa log(P(C))
        
        #   DESDE AQUI SE PLICAN CALCULOS ANTERIORES
        total_palabras_cat = sum(diccionario_cat[categoria].values())
        for palabra in palabras:
            frecuencia = diccionario_cat[categoria][palabra]
            prob_palabra = (frecuencia + 1) / (total_palabras_cat + tam_vocabulario) # suavizado de laplace
            log_prob += math.log(prob_palabra)  # logaritmo de suavizado de laplace
        puntajes[categoria] = log_prob
    print( max(puntajes, key=puntajes.get) ) # imprime la categoria 
    return max(puntajes, key=puntajes.get)   #  Finalmente, selecciona la categora con mayor probabilidad (log-verosimilitud)
